## Project A — ML pre-filter bake-off (do this on the training box)

**Goal:** drive false-positive rate (app says "HOT DOG" on an empty scene) to
near-zero without losing recall on real connectors.

**Why:** classifier is closed-set (6 classes, no "background"), so softmax
always returns something with non-trivial confidence. Hough crop detector +
rembg foreground filter help but get fooled by round objects (cups, knobs,
lights) and textures (wood, fabric). Confirmed in field test on Chris's
iPhone — reported a connector on a no-connector scene.

**Approach:** implement and benchmark three pre-filters server-side, pick the
winner, ship it. Bake-off, not a phased rollout.

### Candidates

1. **CLIP whole-image gate** — cheap and cheerful
   - Score uploaded image against `"a photo of an RF coaxial connector"`
     vs `"a photo with no connector"`
   - If "no connector" wins → return empty predictions, skip the rest of
     the pipeline
   - ~10ms, ~150MB model, runs upstream of `_classify_frame`
   - No training, no bbox

2. **YOLO-World open-vocabulary detector** — replaces Hough
   - Prompt: `"metallic coaxial connector"` (try variants)
   - Returns real bboxes; classifier still does per-class identification
   - ~100-300ms, ~500MB-1GB; smaller/faster than OWL-ViT v2
   - No training

3. **Finetuned YOLOv8n on existing connector data** — proper fix
   - `ultralytics>=8.2.0` already in `training/pyproject.toml`
   - ~13k labeled samples already on disk
   - Convert existing Hough-derived bbox annotations into YOLO format
     (one class: `connector`, or split per-family if you want)
   - Train on the P40 box, evaluate on holdout
   - ~5MB model, ~10ms inference

### Eval set (build before benchmarking)

- **Need negatives** — none in the holdout currently. Collect ~30 phone shots
  of non-connector scenes: cups, knobs, faucets, hands, fabric, wood grain,
  blank desk, lights, lens flares. The harder ones (round shiny objects)
  are the ones that fool the current pipeline.
- **Positives** — start with the 8 holdout shots in `data/test_holdout/`,
  grow toward 30+ via the Flutter contribute screen's "save as test
  holdout" toggle (already wired up).
- Save negatives under `data/test_holdout/_none/` (or wherever fits the
  existing layout).

### Metrics

- **FPR on negatives** — primary kill metric, target ≤ 2%
- **TPR on positives** — must stay ≥ 90%
- Latency per image (ms)
- Model size (MB)

### Concrete steps

1. Collect negatives → `data/test_holdout/_none/` (30 images minimum)
2. Write `scripts/bench_prefilter.py`: takes a dir tree of
   positives + negatives, runs each candidate, dumps a CSV with
   `(image, candidate, accept, conf, latency_ms)`
3. Implement candidates in order of cost: D1 → D2 → D3
4. Score, compare, write up findings in `docs/`
5. Wire winner into `predict_service.py` upstream of `_classify_frame`
6. Smoke test from Flutter app on Chris's iPhone after redeploy
